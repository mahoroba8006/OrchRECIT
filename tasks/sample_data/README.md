## 日付フォーマット仕様

本体アプリの規約に従い、すべての日付列は **ISO 8601 形式（ハイフン区切り）** に統一しています。

| 列 | フォーマット | 例 |
|---|---|---|
| A列 購入日 | `YYYY-MM-DD` | `2025-01-08` |
| H列 取込日時 | `YYYY-MM-DD HH:MM` | `2025-01-08 19:42` |

- 桁数固定（ゼロパディング）・辞書順ソート可能・ロケール非依存
- 本体アプリの `process-receipt` も `toLocaleString('sv-SE')` で同じ形式を出力（2026-05-18 統一）

## Google Sheets へのインポート手順

スクショ撮影用のシートを作る場合の手順:

1. Google Drive で新規スプレッドシート作成（名前は `経費記録_LP用` 等）
2. **A列を選択 → 表示形式 → 数字 → 書式なしテキスト**（重要: 日付の自動変換を防ぐ）
3. `ファイル → インポート → アップロード` で `sample_expense_2025.csv` を選択
4. インポート場所: `現在のシートを置き換える`
5. 区切り文字: `自動検出` (カンマ)
6. **テキストを数値に変換: `いいえ`**（重要: A列が `2025-01-08` のまま保たれる）

### なぜ「いいえ」にするか

Google Sheets は `2025-01-08` のような形式を自動で日付型に変換しようとします。日付型に変換されると Sheets API（`FORMATTED_VALUE`）が返す文字列がロケール依存（`2025/01/08` 等）になり、本体アプリの日付パース処理（`split('-')`）が壊れます。

「いいえ」に設定すれば、A列・H列が文字列リテラルとして保持され、アプリで読み込んだ際も `2025-01-08` 形式のまま取得できます。

### 注意

- アプリ本体の `経費記録.gsheet` には絶対にインポートしないこと（実データが上書きされる）
- スクショ用シートは別ファイルで管理し、撮影後は削除可
- 取込日時列を表示状態にすると履歴っぽさが出るが、画面が狭くなる場合は折り畳み可

撮影候補画面（`2026-05-06` セッション議論で合意済み）:

   - 荷造運賃・諸材料費・農機具費が円グラフで主要科目として並ぶ

   - 「ぶどう関連の支出」「収穫期の出荷費」等で AI 検索デモが映える

4. **Features「自動カテゴリ分類」**: ReviewCard 科目バッジ＋AIコメント
   - 高額品（電動剪定鋏 12.8万円 等）の固定資産候補タグが映える

## 想定外の挙動

このCSVを `経費記録` シートにインポートすると、アプリ側の `migrateColumnsIfNeeded`（H1列ヘッダー検出）は H1=「取込日時」を見て移行不要と判定するため、列追加処理はスキップされる。
